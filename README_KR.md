# DentiCheck: AI 기반 지능형 구강 검진 및 에코시스템

## 1. Inspiration (기획 의도)
DentiCheck 프로젝트의 시작은 글로벌 의료 분야의 고질적인 문제인 **'구강 건강의 낮은 접근성'**에 있었습니다. 충치나 치주 질환은 매우 흔하지만, 많은 사람이 치과 공포증, 높은 비용, 또는 본인의 상태에 대한 정보 부족으로 치과 방문을 미룹니다. 우리는 일반적인 건강 관리가 스마트워치와 스마트폰을 통해 일상화된 것과 달리, 구강 건강은 여전히 '사후 치료' 중심이라는 점에 주목했습니다.

우리는 누구나, 어디서든 스마트폰 카메라만으로 전문적인 수준의 구강 예비 검진을 받을 수 있는 세상을 꿈꾸었습니다. 단순한 '치아 사진' 한 장을 과학적 근거가 담긴 **'구강 건강 인텔리전스 리포트'**로 변환하여, 사용자가 치과를 방문하기 전에 본인의 상태를 정확히 인지하고 행동할 수 있는 지식을 제공하는 것이 우리의 영감이었습니다.

## 2. What it does (주요 기능)
DentiCheck는 원시 데이터를 실제 행동으로 연결되는 의료 지능으로 변환하는 통합 헬스케어 생태계입니다.

*   **AI 시각 진단**: 맞춤형으로 학습된 **YOLOv8** 모델을 사용하여 충치, 치석, 치주염의 징후를 정밀하게 탐지합니다.
*   **지능형 의료 분석 (RAG)**: 단순 탐지를 넘어, **Milvus**에 저장된 방대한 치과 의료 지식 베이스를 실시간으로 검색(RAG)하여, 탐지된 결과에 대해 과학적 근거에 기반한 설명을 제공합니다.
*   **전문적인 자동 리포트 발행**: 시각적 탐지 마커, 위험도 산출 결과(저/중/고), 개인별 관리 계획이 포함된 구조화된 **PDF 리포트**를 생성합니다.
*   **치과 에코시스템 통합**: 진단 결과에 맞는 인근 제휴 치과(카카오 맵 API), 구강 관리 제품, 전용 보험 상품을 매칭합니다.
*   **기업형 운영 관리**: 고성능 페이지네이션이 적용된 어드민 콘솔을 통해 플랫폼 이용 통계 및 제휴사 데이터를 체계적으로 관리합니다.

## 3. How we built it (기술 아키텍처)
본 프로젝트는 AI의 정밀도와 시스템의 확장성을 최우선으로 고려하여 고성능 분산 아키텍처로 설계되었습니다.

### **상세 기술 스택 (Detailed Tech Stack)**

| 계층 (Layer) | 구성 요소 | 사용 기술 (Technologies) |
| :--- | :--- | :--- |
| **Mobile App** | 모바일 인터페이스 | **Expo (React Native)**, TypeScript, React Navigation |
| **Admin Console** | 관리자 대시보드 | **Vite**, **React 18**, Tailwind CSS, Apollo Client |
| **Backend API** | 비즈니스 오케스트레이터 | **Spring Boot 3.2**, Java 17, **GraphQL**, Spring Data JPA |
| **AI Service** | 탐지 및 워크플로우 | **Python 3.11**, **Ultralytics YOLOv8**, **FastAPI**, Kedro |
| **LLM & RAG** | 언어 및 지식 지능 | **LangChain**, **OpenAI**, **Milvus** (Vector DB), PyTorch |
| **Database** | 정형 데이터 관리 | **PostgreSQL 15**, Hibernate |
| **Infrastructure** | DevOps 및 빌드 | **Docker**, Docker Compose, **Gradle**, Poetry |

### **계층별 핵심 역할**
*   **AI 서비스 레이어**: YOLOv8을 이용한 고부하 이미지 분석을 담당하며, **Kedro** 파이프라인을 통해 분석 워크플로우의 안정성을 보장합니다.
*   **백엔드 API 레이어**: 시스템의 '중추'로서 모든 데이터의 흐름을 통제하고, GraphQL을 통해 프론트엔드에 최적화된 데이터를 공급합니다.
*   **프론트엔드 레이어**: 하이브리드 모바일 환경(사용자)과 고성능 웹 대시보드(관리자)를 통해 직관적인 UX를 제공합니다.

## 4. Challenges we ran into (성장 과정의 어려움)
*   **AI 환각 현상 극복**: 생성형 AI의 한계인 '거짓 조언'을 막기 위해 엄격한 **RAG 파이프라인**을 구축했습니다. 모든 설명은 반드시 내부 지식 베이스에서 추출된 근거를 바탕으로만 작성되도록 강제했습니다.
*   **사진 변수 대응**: 모바일 환경의 다양한 조명과 각도를 극복하기 위해 이미지 전처리 기술(Preprocessing)을 고도화하고 학습 데이터의 다양성을 대폭 확장했습니다.
*   **영문 응답 정책 유지**: 사용자가 한국어로 질문하더라도 AI는 반드시 '영문 컨설턴트 모드'를 유지해야 했습니다. 이를 위해 **LlmClient** 수준에서 복합적인 프롬프트 엔지니어링을 수행했습니다.

## 5. Accomplishments (우리의 성과)
*   **전문적인 영문 AI 페르소나**: 인터페이스 언어와 관계없이 일정한 글로벌 전문성을 유지하는 AI 응답 시스템 완성.
*   **완전 자동화 파이프라인**: 사진 촬영부터 의학적 근거가 담긴 PDF 리포트 생성까지 단 10초 이내에 완료되는 프로세스 구현.
*   **확장 가능한 아키텍처**: 백엔드 코드 수정 없이도 AI 모델이나 지식 베이스를 독립적으로 교체/확장할 수 있는 유연한 구조 확보.

## 6. What we learned (배운 점)
*   **맥락의 중요성**: 탐지만으로는 부족하고 설명만으로도 부족합니다. **Vision AI**와 **RAG 기반 LLM**의 시너지가 구강 의료 서비스의 새로운 가치를 창출한다는 것을 배웠습니다.
*   **안정적인 인프라의 가치**: AI 모델이 '두뇌'라면, 백엔드와 데이터베이스는 '신경계'입니다. 탄탄한 아키텍처 없이는 AI의 가치가 사용자에게 전달될 수 없음을 깨달았습니다.

## 7. What's next for DentiCheck (향후 계획)
*   **AR 촬영 가이드**: 촬영 시 실시간 가이드를 표시하여 사용자가 완벽한 각도의 사진을 찍을 수 있도록 지원할 예정입니다.
*   **동영상 스캔**: 사진을 넘어 짧은 영상을 기반으로 360도 구강 구조를 분석할 계획입니다.
*   **병원 EMR 연동**: 생성된 PDF 리포트를 치과 병원의 전자의무기록(EMR) 시스템과 직접 연동할 예정입니다.
