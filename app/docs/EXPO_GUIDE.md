# Expo란 무엇인가? (입문자 가이드)

## 1. Expo란?

**Expo**는 **React Native**를 기반으로 한 프레임워크이자 플랫폼입니다.  
React Native가 "웹 개발 방식으로 네이티브 앱을 만드는 도구"라면, Expo는 "그 도구를 **더 쉽고, 빠르고, 편리하게** 사용할 수 있도록 돕는 **종합 세트(Toolchain)**"라고 할 수 있습니다.

웹 개발에서의 `Create React App`이나 `Next.js`와 비슷한 역할을 한다고 이해하면 쉽습니다. 복잡한 네이티브 설정(Xcode, Android Studio) 없이 JavaScript/TypeScript 만으로 앱을 개발, 빌드, 배포할 수 있게 해줍니다.

---

## 2. 왜 Expo를 사용하는가? (핵심 기능)

React Native CLI(Pure React Native)를 사용할 때 겪는 가장 큰 어려움은 **네이티브 개발 환경 세팅**과 **모듈 연동**입니다. Expo는 이 문제를 해결해 줍니다.

*   **빠른 시작**: 복잡한 설치 과정 없이 `npx create-expo-app` 명령어 하나로 개발 준비가 끝납니다.
*   **Expo Go 앱**: 개발 중인 앱을 스마트폰에 설치 과정 없이 `QR 코드`만 찍어서 바로 실행해 볼 수 있습니다. (Mac 없이도 윈도우에서 아이폰 테스트 가능)
*   **강력한 SDK**: 카메라, 위치 정보, 알림, 센서 등 자주 쓰는 네이티브 기능들이 미리 통합되어 있어 쉽게 가져다 쓸 수 있습니다.
*   **EAS (Expo Application Services)**: 클라우드 빌드 및 배포 시스템을 제공하여, 로컬 컴퓨터 성능에 구애받지 않고 앱스토어/플레이스토어에 배포할 수 있습니다.

---

## 3. Expo의 장단점

### ✅ 장점 (Pros)

1.  **개발 속도 및 생산성 향상**
    *   초기 설정 시간이 거의 0에 가깝습니다.
    *   iOS/Android 코드를 따로 건드릴 필요가 거의 없습니다.

2.  **OS 제약 없는 개발 환경**
    *   Windows 사용자도 Expo Go를 통해 iOS 앱을 테스트할 수 있습니다. (단, 최종 빌드는 EAS Build 사용 필요)

3.  **OTA (Over The Air) 업데이트**
    *   앱스토어 심사 없이 JavaScript 코드 변경 사항(버그 수정, UI 변경 등)을 사용자에게 즉시 배포할 수 있습니다. (`expo-updates`)

4.  **쉬운 유지보수**
    *   네이티브 모듈 버전 관리의 복잡함을 Expo SDK가 대신 관리해 줍니다.

### ❌ 단점 (Cons)

1.  **네이티브 모듈 사용의 제약 (과거의 단점)**
    *   과거에는 Expo SDK에 포함되지 않은 네이티브 라이브러리를 사용할 수 없었습니다.
    *   **현재:** **Development Builds (Dev Client)** 와 **Config Plugins**의 도입으로 이 단점은 거의 해결되었습니다. 이제 커스텀 네이티브 코드도 사용 가능합니다.

2.  **앱 용량 (App Size)**
    *   편리한 기능들이 많이 내장되어 있어, "Hello World" 앱조차도 Pure React Native보다 용량이 조금 더 큽니다. (수십 MB 수준 차이)

3.  **유료 서비스 (EAS)**
    *   EAS Build 및 Update의 무료 플랜은 제한이 있어, 규모가 커지면 비용이 발생할 수 있습니다.

---

## 4. 언제 Expo를 써야 할까?

### 👍 추천하는 경우
*   **빠르게 MVP(최소 기능 제품)를 만들어야 하는 스타트업/해커톤**
*   네이티브 개발(Swift, Kotlin) 경험이 없는 웹 개발자
*   Windows 환경에서 iOS/Android를 동시에 개발하고 싶은 경우
*   표준적인 기능(카메라, 지도, 로그인 등) 위주의 앱

### 👎 비추천하는 경우 (매우 드묾)
*   OS의 매우 깊숙한 저수준(Low-level) API를 직접 건드려야 하는 경우 (예: 블루투스 저전력 프로토콜의 비표준 확장 등)
*   앱 용량이 1MB라도 작아야 하는 초경량 앱이 필요한 경우

---

## 5. 결론

**"일단 Expo로 시작하세요."**

현재 React Native 커뮤니티와 공식 문서에서도 **Expo 사용을 권장**합니다.  
과거의 단점이었던 확장성 문제도 **Prebuild**와 **Dev Client** 시스템으로 해결되었기 때문에, 굳이 처음부터 복잡한 React Native CLI로 시작할 이유는 거의 없습니다. DentiCheck 프로젝트 역시 개발 효율성을 위해 Expo를 선택한 것은 훌륭한 결정입니다.
